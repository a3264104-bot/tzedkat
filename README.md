# צדקת רבותינו — אתר הזמנות

אתר הזמנות בעברית (RTL) לעסק **"צדקת רבותינו — עופות, בשר ודגים לכבוד שבת ויום טוב"**.
מכירות תקופתיות לפי נקודות חלוקה ותאריכי חלוקה, עם אזור ניהול מלא.

הכל ניתן לעריכה מתוך מערכת הניהול — אין מחירון קשיח בקוד. המחירון מה‑PDF יובא רק כ"זרע" התחלתי דרך ה‑seed.

## טכנולוגיה

- **Next.js 15** (App Router) + **TypeScript**
- **Tailwind CSS** — RTL מלא, מובייל קודם
- **Prisma** + **PostgreSQL**
- **Auth.js v5** (next-auth) — התחברות מנהל מוגנת בסיסמה
- **xlsx** — ייצוא לאקסל

## דרישות מקדימות

- Node.js 18.18+ (מומלץ 20+)
- מסד נתונים PostgreSQL (מקומי, או Supabase / Neon / Vercel Postgres)

## התקנה והרצה מקומית

```bash
# 1. התקנת חבילות
npm install

# 2. הגדרת משתני סביבה
cp .env.example .env
#   ערוך את .env והגדר:
#   DATABASE_URL  — מחרוזת החיבור ל‑PostgreSQL
#   AUTH_SECRET   — מחרוזת אקראית ארוכה (אפשר ליצור עם: npx auth secret)
#   ADMIN_EMAIL / ADMIN_PASSWORD — פרטי המנהל הראשוני (ל‑seed)

# 3. יצירת הסכימה במסד הנתונים
npm run db:push

# 4. ייבוא נתונים ראשוניים (מנהל, קטגוריות, מוצרים, נקודות חלוקה, מחירון תמוז תשפ"ו)
npm run db:seed

# 5. הרצה
npm run dev
```

האתר יעלה בכתובת http://localhost:3000

## כניסת מנהל

- כתובת: `/admin`
- מייל: הערך מ‑`ADMIN_EMAIL` (ברירת מחדל `admin@tzedkat.co.il`)
- סיסמה: הערך מ‑`ADMIN_PASSWORD` (ברירת מחדל `admin1234`)

> ⚠️ יש לשנות את הסיסמה לפני עלייה לאוויר.

## תהליך ההזמנה (לקוח)

1. עמוד בית עם לוגו וכפתור "להזמנה".
2. בחירת נקודת חלוקה.
3. הצגת פרטי החלוקה (כתובת, תאריך, שעה, נציג, טלפון) + **אישור חובה** שראה את התאריך והשעה.
4. בחירת מוצרים וכמויות (כולל אפשרות "בודדים" בתוספת מחיר לק"ג).
5. סל הזמנה + מילוי פרטי לקוח (שם, טלפון, טלפון נוסף, הערות).
6. מסך סיכום עם הבהרת **מחיר משוער** ואזהרת **תשלום במזומן (אין בהקפה)** + **אישור חובה**.
7. שליחה וקבלת אישור.

המחירים מחושבים **מחדש בשרת** בעת שליחת ההזמנה — הלקוח לא יכול לזייף סכומים.

## כללי הזמנה ואכיפה

המערכת אוכפת את חוקי ההזמנה גם בצד הלקוח וגם **שוב בשרת** (אי אפשר לעקוף):

- כל הזמנה משויכת למכירה (מחירון) ולנקודת חלוקה — אין הזמנה ללא הקשר.
- ניתן להזמין רק ממכירה בסטטוס **פעיל** ובתוך חלון הזמן (אחרי `openDate`, לפני `closeDate`).
- אם עברה שעת הסגירה — השרת דוחה את ההזמנה והאתר מציג "מועד ההרשמה הסתיים".
- נקודת החלוקה חייבת להשתתף במכירה; אחרת ההזמנה נדחית.
- מוצרים שאינם פעילים או שאינם משתתפים במכירה לא מוצגים ולא ניתנים להזמנה.
- המחיר נשמר ב‑snapshot ברמת הפריט (`unitPrice`, `estimatedPrice`) וברמת ההזמנה
  (שם הנקודה, תאריך החלוקה ושם המכירה כפי שהוצגו ללקוח). שינוי מחיר עתידי **לא** משפיע
  על הזמנות קיימות — רק עריכה ידנית מפורשת של המנהל משנה הזמנה.
- המחיר הסופי (`finalPrice` / `finalTotal`) נקבע לפי המשקל בפועל ויכול להיות שונה מהמשוער.

### כמות מוגבלת

למוצר אפשר להגדיר "כמות מוגבלת" ולציין **מגבלת כמות** (שדה למנהל). זו אזהרה בלבד —
**לא חוסמת לקוחות**. בעמוד הדוחות מוצגת אזהרה כשסך ההזמנות של מוצר מתקרב למגבלה (80%+)
או חורג ממנה.

## מחיר קרטון / בודדים

בבשר ודגים המחיר במחירון הוא בקרטון. הלקוח יכול לבחור "בודדים" ואז מתווספת תוספת לק"ג
(ברירת מחדל 3 ₪, נקבעת ברמת המחירון וניתנת לעריכה במערכת הניהול, וגם פר‑מוצר).

## דוחות וייצוא

- דוח לפי נקודת חלוקה
- סיכום מוצרים להכנה (כמה ק"ג / יחידות מכל מוצר)
- דוח לקוחות
- דוח כספי (מכירות לפי מכירה / נקודה / מוצר, נמסרו, בוטלו)
- ייצוא לאקסל + הדפסה

## העלאה לאוויר (Vercel + Supabase)

1. צור פרויקט ב‑Supabase וקח את מחרוזת החיבור (Connection string, מצב "Session" / pooler).
2. דחוף את הקוד ל‑GitHub וחבר אותו ל‑Vercel.
3. ב‑Vercel הגדר משתני סביבה: `DATABASE_URL`, `AUTH_SECRET`, `ADMIN_EMAIL`, `ADMIN_PASSWORD`.
4. הרץ פעם אחת מקומית (מול ה‑DATABASE_URL של הפרודקשן) את:
   ```bash
   npm run db:push
   npm run db:seed
   ```
   או דרך Supabase SQL / מסוף. ה‑seed יוצר את המנהל, המחירון והנתונים הראשוניים.
5. Deploy. סקריפט ה‑build כולל `prisma generate` אוטומטית.

## הערות

- כדי **לרענן** את המחירון הפעיל אחרי seed חוזר — שים לב ש‑seed מוסיף נתונים; להרצה נקייה אפס/אתחל את ה‑DB תחילה.
- לוגו: ברירת המחדל היא SVG מוטמע (`src/components/Logo.tsx`). הקובץ המקורי נמצא ב‑`public/logo.png` למקרה שתעדיף להציג אותו.
- סליקה אונליין לא נדרשת בשלב זה — ההזמנה נשלחת למנהל והתשלום מתבצע מול הנציג.
