# בדיקות לוגיקה (end-to-end על הקוד האמיתי)

הבדיקות מריצות את **קוד ה-route האמיתי** (orders / reports / export) מול מסד נתונים
בזיכרון בצורת Prisma, ללא צורך ב-PostgreSQL או ב-Prisma engine.

## הרצה
```bash
cd tests
npm init -y
npm install esbuild@0.24 zod@3 xlsx@0.18 pg-mem@3
# מעתיק את קבצי ה-route מהפרויקט (מתבצע ע"י build.mjs דרך נתיבים יחסיים)
node build.mjs   # מתרגם את ה-routes
node test.mjs    # מריץ 38 בדיקות
```

> הערה: build.mjs מצפה לקבצי המקור `_orders_route_src.ts` וכו'. כדי להריץ מהמאגר,
> העתק את שלושת ה-routes לתיקיית tests עם השמות הללו, או עדכן את הנתיבים ב-build.mjs.

## מה נבדק (38 בדיקות, כולן עוברות)
1-3. טעינת מכירה פעילה, נקודות, תאריך חלוקה, ואכיפת אישור-תאריך ב-UI
4-5. ארבעה סוגי מוצרים + חישוב סכום בשרת (כולל תוספת בודדים) + מניעת זיוף מחיר
6. חסימת הזמנה לפני פתיחה / אחרי סגירה / מכירה לא פעילה
7. חסימת מוצר לא פעיל
8. חסימת נקודה שלא משתתפת במכירה
9. orderNumber ייחודי + snapshot מלא (נקודה/תאריך/מכירה/מחיר)
10. הזמנה מופיעה ב-GET מנהל ובדוחות (+401 ללא הרשאה)
11. אזהרת כמות מוגבלת מעל 80% / חריגה / ללא אזהרה מתחת ל-80%
12. ייצוא אקסל בעברית + RTL תקין (rightToLeft נכתב לקובץ)
